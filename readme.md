# Wildberries Parser

Проект выполнен для демонстрации работы с парсингом.
Назначение проекта — собирать информацию о товарах из **Wildberries** по поисковому запросу и показывать ввиде таблицы на экране.

Каждая строка таблицы содержит следующую информацию:

* Артикул
* Источник
* Название
* Цена
* Картинка
* Название селлера
* Остатки по товару
* Рейтинг
* Количество отзывов

---

# Общее техническое описание

Проект выполнен в **объектном стиле** и состоит из нескольких множеств объектов. Каждое из множеств объектов имеет:

* **контракт (интерфейс)** — находится в файле, названном **во множественном числе** от имени объекта;

* **чистую логику без побочных эффектов** — находится в файле, названном **в единственном числе** от имени объекта.

# Принципы, применяемые при проектировании

## 1. Отложенные вычисления

Используются для экономии ресурсов и быстрой инициализации объектов.

Часть объектов может быть **частично ленивой**. Например, товар может получать `id` или `name` при инициализации, а более крупные данные загружать только при необходимости.

Конструкторы объектов вынесены в **статические методы**, так как в Python отсутствует возможность перегрузки конструкторов.

---

## 2. Минимальная проверка при инициализации

Для обеспечения ленивости при создании объекта выполняются только **критические проверки**.

Остальная валидация производится **по мере загрузки данных** в других методах.

---

## 3. Один объект — одна ответственность

Весь функционал распределён по **обёрткам и декораторам** относительно "корневого" объекта.

Каждый новый декоратор добавляет свою часть функциональности:

* логирование
* получение данных из источника
* кэширование
* и т.д.

**Корневой объект** является носителем **чистых вычислений (бизнес-логики)**.

### Пример модуля `product`

```
products.py             — контракт и структура данных
product.py              — бизнес-логика
wildberries_product.py  — получение данных с Wildberries
json_cached_product.py  — кэширование данных в файл
```

Такой подход позволяет:

* создавать разные конфигурации (**production**, **тестирование**);
* заменять источник данных без переписывания вычислений
  (например: **Wildberries**, **Ozon** и др.).
